# Cricify Stremio Addon - Agent Guidelines

## Overview
This repository contains a FastAPI-based Stremio addon that scrapes live TV and sports events from a custom provider (Cricify). It uses Firebase Remote Config to resolve dynamic API URLs and performs AES decryption on the encrypted API payloads.

## Codebase Structure & Responsibilities
- **`main.py`**: The Stremio addon HTTP interface (FastAPI). Contains the `MANIFEST` and standard Stremio endpoints: `/catalog`, `/meta`, and `/stream`. Converts internal API JSON into Stremio's expected format.
- **`cricify.py`**: The data retrieval layer. Fetches encrypted lists of live events and stream details from the provider. Also handles parsing of URLs and custom HTTP headers.
- **`firebase.py`**: Interacts with the Firebase Remote Config API mimicking an Android client to fetch the dynamic API base URL (`cric_api1` or `cric_api2`).
- **`crypto.py`**: Handles AES-CBC decryption of the data payloads returned by the provider.
- **`config.py`**: Manages environment variables required for Firebase and Crypto keys.

## Data Flow
1. **API URL Resolution**: `firebase.py` fetches the latest active API endpoint from Firebase.
2. **Catalog/Meta Fetching**: Stremio requests catalog -> `main.py` calls `cricify.fetch_live_events()` -> downloads `live_events.txt` -> `crypto.py` decrypts it.
3. **Stream Resolution**: Stremio requests stream -> `main.py` calls `cricify.fetch_channel_streams(slug)` -> downloads `channels/{slug}.txt` -> decrypts it -> extracts URLs and headers -> maps to Stremio behavior hints.

## Future Enhancements & Where to Implement Them

### 1. Advanced DRM / ClearKey Support
**Where**: `main.py` -> `get_stream()`
**What to do**: Currently, streams with `type == "7"` (DRM/ClearKey) are parsed but cannot be directly played in Stremio Web. If a custom Stremio player integration, proxy, or external player intent is needed to support DRM streams, modify the logic inside `get_stream` where the `api` key is split into `drm_kid_hex` and `drm_key_hex`.

### 2. Updating Firebase App Properties
**Where**: `firebase.py` -> `fetch_remote_config()`
**What to do**: The provider might block requests if the Firebase payload contains outdated Android application versions. Update `appVersion`, `appBuild`, and `sdkVersion` in the payload dictionary to match the latest Android app release if API requests start failing.

### 3. Adding New Categories / VOD Support
**Where**: `main.py` -> `MANIFEST` and Route Handlers
**What to do**: Update the `MANIFEST` dictionary to include new `catalogs` or `types` (e.g., `"movie"`). Implement the corresponding `@app.get("/catalog/...` and `@app.get("/meta/...` functions, and add new data retrieval methods in `cricify.py`.

### 4. Updating Decryption Keys or Algorithms
**Where**: `crypto.py` and `.env` / `config.py`
**What to do**: If the provider changes how secrets are structured (currently hex strings separated by `:` for Key and IV), update `_parse_key_info()` in `crypto.py`. If they switch from AES-CBC to another algorithm or use different padding, update `_try_decrypt()`. Ensure new keys are added to `.env`.

### 5. Custom Proxy for Headers/Stream Routing
**Where**: New file (e.g., `proxy.py`) & `main.py`
**What to do**: Some Stremio clients don't fully support custom HTTP headers (like `Referer` or `User-Agent`) sent via `behaviorHints`. If streams fail to play, implement a lightweight streaming proxy endpoint within the FastAPI app that streams chunks from the original source while applying the headers, and point Stremio to this local proxy URL instead.
